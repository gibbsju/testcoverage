Context and Methodology

Try push environment: VirtualMac2,1, 8GB RAM, 4 cores — matches target Tart spec (16GB Mac Mini / 2 VMs).

Classification (validated):

  • Verified: Suite passed on the try push VM. Ready for production VM rollout.
  • Fixable: Partially working. Known fixes: timeout tuning for VM slowness, skip window-focus tests, update test expectations, fix packaging bugs.
  • Blocked: Real VM-specific issues. The biggest blocker is the browser-chrome / reftest harness hang — Firefox starts, Marionette connects, then nothing for 1000s. This is macOS VM-specific (works on Linux VMs) and affects the largest cluster of suites. Resolving this single issue would unblock crashtest, mochitest-browser-chrome, browser-a11y, browser-translations, browser-media, reftest, and jsreftest.
  • Confirmed bare metal: Try push confirms these crash or fail fundamentally on VMs (gtest SIGSEGV, xpcshell crash recovery).
  • Not tested: Config analysis suggests these may be VM-capable but they weren't in the try push. Needs future validation.

Key finding from try push: Config-based analysis (virtualization: virtual on Linux/Windows) correctly predicts resource/GPU compatibility but cannot detect macOS VM-specific runtime behaviors like the harness hang and window focus issues. Real try push validation is essential.

Positive surprise: Tart VMs have working GPU — mochitest-plain-gpu, WebGL extension tests, encrypted media, and print rendering all passed. Several suites originally classified as "Medium" were promoted to "Verified."
